# workspacecord 官网设计稿

**日期：** 2026-04-02
**目标目录：** `/Users/ld/Documents/github/agentcord/site`

## 目标

为 `workspacecord` 设计并实现一个未来感、极客终端感的产品官网，用来清晰演示其核心工作流：**本地终端 -> Discord -> 主代理 -> 子代理 -> 历史归档**。

官网的首要目标不是完整文档化所有能力，而是：

1. 让首次访问者一眼理解产品是什么。
2. 让目标用户感受到工作流的独特性与控制感。
3. 将用户转化到“快速开始”。

## 目标受众

优先受众为：**极客 / 效率工具用户**。

这些用户通常：
- 喜欢命令行和本地优先工具。
- 对自动化、代理协作、终端工作流敏感。
- 愿意尝试结构明确、可组合、可追踪的新工具。

## 品牌与语气

- 品牌名：`workspacecord`
- 风格关键词：未来感、终端感、控制台、克制、清晰、可运行
- 避免：泛 SaaS 感、企业宣传腔、卡片堆砌、花哨特效

## 页面定位

官网应当被设计成一个**多智能体工作流发布页**，而不是普通营销页。

核心表达：

> 把本地终端和 Discord 连成一个多智能体开发控制台。

目标：
- 立即建立品牌识别。

## 首屏主视觉设计

主视觉必须是一个**可交互网页动画**，直接演示完整工作流，而不是静态插图。

### 演示内容

需要表现的流转关系：

1. 用户在本地终端运行命令。
2. 项目被挂载并绑定到 Discord 服务器。
3. 控制频道生成主代理会话。
4. 主会话派发多个子代理线程。
5. 子代理执行任务、更新状态、等待审批或完成。
6. 结果归档到 `#history`。

### 交互方式

推荐：**自动播放 + 悬停高亮**

- 页面加载后自动演示一轮完整流程。
- 用户悬停任一步骤时：
  - 当前步骤高亮。
  - 主视觉对应区域同步高亮。
  - 旁边出现该步骤的简短说明。

### 画面分区

- 左：本地终端面板
- 中：Discord 服务器 / 频道 / 线程结构
- 右：主代理与子代理执行状态
- 底 / 尾：归档状态与历史记录

## 视觉规范

### 颜色
- 背景：近黑冷调
- 主文字：高亮冷白
- 次文字：低对比灰蓝
- 主强调：Discord 紫
- 状态辅助：终端荧绿

### 字体
- 标题：可带轻微终端 / 展示字体个性
- 正文：清晰无衬线
- 命令、路径、状态：等宽字体

### 动效
仅保留三类：
- 首屏分层淡入
- 状态点 / 游标 / 连线轻微脉冲
- 滚动显现

避免重特效和复杂滚动绑定。

## 技术方向

建议新建独立站点目录：`site/`

优先考虑：
- `Next.js` 或 `Vite + React`
- `Tailwind CSS`
- `Framer Motion`

实现重点：
- 可维护的单页发布页
- 首屏工作流动画组件化
- 响应式布局
- 快速本地构建和部署

## 成功标准

如果实现完成，页面应满足：

1. 用户在首屏 5 秒内理解产品作用。
2. 工作流动画能明确展示“本地终端 -> Discord -> 代理 -> 归档”的关系。
3. 页面视觉气质明显偏未来感终端 / 控制台，而不是普通 SaaS 模板。
4. “快速开始”按钮成为主转化动作。
